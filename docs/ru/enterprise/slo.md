# Целевые показатели уровня обслуживания (SLO) и бюджет ошибок

В этом документе изложены целевые показатели эксплуатационной надежности (SLO) и бюджеты ошибок (Error Budgets) для уровня SGR Kernel V3 Enterprise Tier. Он служит базовым контрактом (SLA) для оценки производительности и доступности системы.

## 1. Доступность (Availability)

**Цель**: API ядра SGR и оркестратор Swarm должны быть доступны для приема и постановки в очередь запросов 99,9% времени.

- **SLI (Индикатор)**: Процент ответов HTTP 200/202 для эндпоинтов `/health/swarm_topology` и `/execute` за скользящее 30-дневное окно.
- **Error Budget (Бюджет ошибок)**: 0,1% (примерно 43 минуты допустимого простоя в месяц).
- **Последствия исчерпания бюджета**: Остановка разработки фич (Feature Freeze). Если бюджет ошибок исчерпан, слияние всех новых функций приостанавливается, и 100% инженерных усилий перенаправляется на обеспечение надежности и сокращение технического долга до восстановления бюджета.

## 2. Задержка (Latency)

*Примечание о происхождении: Конкретные пороговые значения задержки, указанные ниже, были установлены во время наших бенчмарков нагрузочного тестирования Locust на Фазе 6 (проверка слоя Swarm Routing при высокой конкурентности).*

**Цель 1 (Маршрутизация)**: Основная система (Маршрутизация, Бюджетирование, Комплаенс-проверки) добавляет минимальные накладные расходы.
- **SLO**: p95 задержка < 500мс для инициализации оркестратора Swarm и предполётных проверок (pre-flight checks).

**Цель 2 (Инференс LLM)**: Вызовы облачных LLM зависят от очередей провайдеров, но локальные резервные модели (fallback) обеспечивают гарантированные временные рамки для критических путей выполнения.
- **SLO**: p95 Задержка < 2.0с до появления первого токена (TTFT - Time-To-First-Token) для моделей уровня `secure_local`.

## 3. Корректность и Комплаенс (Correctness & Compliance)

**Цель**: Формальные инварианты (TLA+) и спецификации вывода (`OutputSpec`) строго соблюдаются.
- **SLO**: 100% соответствие определенным проверкам `OutputSpec`. 0% утечек формально запрещенных PII-токенов (персональных данных).
- **Error Budget**: 0,0%. Единичное нарушение формального инварианта, пробившее границу системы, считается инцидентом уровня SEV-1, требующим немедленного проведения RCA (Root Cause Analysis).

## 4. Правила оповещений (Alerting Rules для Prometheus/Grafana)
Стандартные шаблоны предоставляются в конфигурации репозитория `config/prometheus.yml`:
- **`Alert: HighLatency`**: Срабатывает, если p95 TTFT > 2с в течение 5 последовательных минут.
- **`Alert: ModelHandoffFailing`**: Срабатывает, если `ModelRouter` не может найти рабочий маршрут (статус `up`) и вынужденно переключается на `fallback-none` более 5 раз за 10 минут. 
- **`Alert: BudgetGuardBreachAttempt`**: Срабатывает, если какой-либо агент попытался обойти блокировку `TokenLedger` при исчерпанном бюджете.
