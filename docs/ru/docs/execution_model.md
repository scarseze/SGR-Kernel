# Модель выполнения SGR Kernel Swarm (v3.x)

## Обзор
Ядро SGR Kernel выступает в роли детерминированного **Слоя Корректности (Correctness Layer)** для автономных мультиагентных роев (swarms). В отличие от традиционных исполнителей DAG, которые следуют статически заданному пути, модель выполнения Swarm фокусируется на **динамической пошаговой оркестрации**, автономных передачах (handoffs) и формальных инвариантах безопасности.

## 1. Цикл Swarm
Выполнение следует рекурсивному циклу **Промпт -> Выполнение -> Маршрутизация**:

1.  **Запрос**: Получение ввода пользователя и начального контекста.
2.  **Инициализация хода**: `SwarmEngine` инициализирует активного агента (по умолчанию `RouterAgent`).
3.  **Цикл выполнения**:
    - **Генерация контекста LLM**: Активный агент обрабатывает историю и генерирует ответ или вызов инструмента.
    - **Выполнение возможностей**: Навыки (инструменты) вызываются через `SkillLifecycleEngine`.
    - **Динамическая маршрутизация (Handoff)**: Если агент решает передать управление, генерируется событие `TransferToAgent`.
4.  **Верификация**: `CriticPolicy` проверяет результат. В случае отказа запускается повторная попытка или эскалация к человеку.
5.  **Контрольные точки (Checkpointing)**: После каждой мутации состояния (хода) `StateManager` сохраняет `ExecutionState` через Redis/SQLite.

## 2. Конечный автомат Swarm
Состояние выполнения проходит через динамические фазы:

| Состояние | Описание |
| :--- | :--- |
| `ACTIVE` | Рой в процессе обработки хода. |
| `WAITING_FOR_INPUT` | Выполнение приостановлено в ожидании ответа пользователя. |
| `PAUSED_APPROVAL` | Human-in-the-Loop: приостановка из-за отказа критика или флагов безопасности. |
| `COMPLETED` | Рой завершил ход или сгенерировал финальный ответ. |
| `FAILED` | Критическое завершение (например, превышен бюджет или лимит ходов). |
| `ABORTED` | Принудительная остановка политикой безопасности или пользователем. |

## 3. Потоки данных и передачи (Handoffs)
- **Контекст хода**: История передается между агентами, служа "общей памятью" роя.
- **Handoffs**: Автономная передача управления, не требующая предопределенного графа (нет накладных расходов DAG).
- **Санитизация**: `ContextSanitizer` скрывает чувствительные данные (PII) перед передачей контекста между агентами или пользователю.

## 4. Формальные гарантии
| Гарантия | Механизм |
| :--- | :--- |
| **Атомарная видимость** | Результаты видны пользователю только после успешного маркера фиксации состояния. |
| **Лимит ходов** | Ограничение `max_turns` предотвращает бесконечные циклы перенаправлений. |
| **Защита PII** | `ContextSanitizer` на базе регулярных выражений фильтрует токены при передачах. |
| **Восстановление** | Автоматическое возобновление с последней точки (Turn Checkpoint) при сбоях инфраструктуры. |
