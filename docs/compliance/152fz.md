# 🇷🇺 152-ФЗ Compliance (О Защите Персональных Данных)

SGR Kernel предоставляет механизмы для полного соответствия требованиям Федерального закона № 152-ФЗ «О персональных данных» при разработке агентов и RAG-систем.

## 🛡️ Основные гарантии

1. 🌍 **Локализация данных (ст. 18 152-ФЗ)**
   Система позволяет жестко привязать обработку запросов и хранение векторных баз знаний (Milvus/PostgreSQL) исключительно к дата-центрам, расположенным на территории РФ (например, `ru-central1`).
   
2. 🎭 **Маскирование PII в реальном времени**
   Перед отправкой контекста в LLM встроенный `SecurityGuardian` паттерн-матчингом находит и анонимизирует российские форматы данных: PASSPORT_RF, INN, SNILS, BIK.

3. 📜 **Защищенный аудиторский след**
   Все цепочки рассуждений (Chain-of-Thought) агентов пишутся в неизменяемый WAL с ГОСТ-хешированием. Это гарантирует прозрачность для аудита Роскомнадзором без риска утечки логов ИСПДн.

4. 🔒 **Интеграция с ГОСТ криптографией**
   SGR Kernel поддерживает настройку криптопровайдеров (CryptoPro, ViPNet) для шифрования данных "в покое" и TLS-соединений между микросервисами по ГОСТ Р 34.12-2015.

**Пример конфигурации (`configs/compliance/rf_152fz_example.yaml`):**
```yaml
compliance_level: rf_152fz

data_localization:
  primary_storage_region: "ru-central1"
  backup_storage_region: "ru-central1"

pii_patterns:
  passport_rf: "\\b\\d{4}\\s?\\d{6}\\b"
  inn_individual: "\\b\\d{12}\\b"
  snils: "\\b\\d{3}-\\d{3}-\\d{3}\\s?\\d{2}\\b"
```

## 🇺🇸 English

# 🇷🇺 152-FZ Compliance (Russian Personal Data Law)

SGR Kernel provides mechanisms for full compliance with the Federal Law No. 152-FZ "On Personal Data" when developing AI agents and RAG systems for the Russian market.

## 🛡️ Core Safeguards

1. 🌍 **Data Localization (Art. 18 of 152-FZ)**
   The system allows administrators to strictly bind query processing and vector knowledge base (Milvus/PostgreSQL) storage exclusively to data centers located within the Russian Federation (e.g., `ru-central1`).
   
2. 🎭 **Real-time PII Masking**
   Before sending context to an LLM, the built-in `SecurityGuardian` uses pattern matching to find and anonymize Russia-specific data formats: PASSPORT_RF, INN, SNILS, BIK.

3. 📜 **Secure Audit Trail**
   All agent Chain-of-Thought reasoning is written to an immutable WAL with GOST hashing. This guarantees transparency for Roskomnadzor audits without risking PDIS (Personal Data Information System) log leaks.

4. 🔒 **GOST Cryptography Integration**
   SGR Kernel supports configuring crypto-providers (CryptoPro, ViPNet) for "data at rest" encryption and internal microservice TLS connections in accordance with GOST R 34.12-2015.

**Configuration Example (`configs/compliance/rf_152fz_example.yaml`):**
```yaml
compliance_level: rf_152fz

data_localization:
  primary_storage_region: "ru-central1"
  backup_storage_region: "ru-central1"

pii_patterns:
  passport_rf: "\\b\\d{4}\\s?\\d{6}\\b"
  inn_individual: "\\b\\d{12}\\b"
  snils: "\\b\\d{3}-\\d{3}-\\d{3}\\s?\\d{2}\\b"
```
