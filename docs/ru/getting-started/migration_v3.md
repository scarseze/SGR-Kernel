# Справочник по миграции: SGR Kernel V2 → V3

Это руководство описывает процесс перехода корпоративных клиентов с версии SGR Kernel V2 (Monolith/Single-Agent) на V3 (Multi-Agent Swarm Enterprise Tier).

## Ключевые архитектурные изменения

1. **Разделенное управление состоянием (Decoupled State Management)**
   - V2 полагался на локальную память и SQLite.
   - V3 использует централизованное состояние **SwarmState** на базе Redis для синхронизации нескольких агентов и распределенных блокировок.
   - *Действие*: Убедитесь, что `redis-server` доступен и `REDIS_URL` настроен в вашем окружении (или поднимите предоставляемый `docker-compose.yml`).

2. **От одного агента к движку SwarmEngine**
   - В V2 использовался линейный путь выполнения (`Kernel.run()`).
   - В V3 представлен `SwarmEngine`, который динамически маршрутизирует задачи специализированным агентам (`CriticEngine`, `Researcher` и др.) через топологии направленного ациклического графа (DAG).
   - *Действие*: Обновите устаревшие вызовы `Kernel.run()`, инициализируйте `SwarmEngine` и отправляйте планы через `orchestrator.execute()`.

3. **Корпоративные средства защиты (Новое в V3)**
   - **Economic Layer (Экономический слой)**: Все вызовы LLM теперь перехватываются `BudgetGuard`. Вы должны явно предоставить `economic_budget` в контексте сессии, иначе выполнение завершится ошибкой `BudgetExceededError`.
   - **Compliance Engine (Комплаенс)**: Маршрутизация данных теперь проверяется на соответствие географическим и правовым ограничениям (например, `152-FZ`). При обработке конфиденциальных данных (PII) система требует маршрута `secure_local` (онпремис-модели).

## Пошаговая миграция

1. **Обновление инфраструктуры**
   Запустите новый мастер быстрого старта (развернет Redis, Jaeger, Qdrant):
   ```bash
   pwsh ./scripts/quickstart.ps1
   ```

2. **Обновление зависимостей в коде**
   V3 вводит строгие проверки безопасности при выполнении графа.
   ```python
   # Старый подход V2
   response = kernel.execute("Исправь мой код")
   
   # Новый подход V3 с защитными механизмами
   from core.swarm import SwarmEngine
   
   engine = SwarmEngine()
   # Задаем ограничения безопасности и бюджета
   context = {
       "economic_budget": 0.50, # Максимальный бюджет в USD
       "requires_local": True   # Принудительное использование локальной модели (ModelRouter)
   }
   response = await engine.execute(plan, context=context)
   ```

3. **Обработка нарушений верификации**
   V3 применяет строгие лимиты `OutputSpec`. Если ваши downstream-сервисы ожидают определенные JSON-схемы, зарегистрируйте их для агента перед выполнением, чтобы включить автоматическое самовосстановление (Self-Healing) при галлюцинациях LLM.
